# Example Use Cases

## Use case 1: Cloud connectivity option review

### Situation
A team is evaluating multiple connectivity approaches between cloud providers and wants to understand networking tradeoffs, hidden costs, operational burden, and likely project risks.

### Useful workflows
- Context Sharding
- Complexity Mapper
- Decision Brief

### Desired output
- hidden risks
- complexity heat map
- dependency summary
- recommended next checks

## Use case 2: Vendor documentation risk scan

### Situation
A vendor pitch sounds clean, but the team needs help finding buried caveats, quotas, region limitations, operational constraints, and support burden.

### Useful workflows
- Context Sharding
- caveat-extractor
- cost-capacity-analyst
- Complexity Mapper

### Desired output
- caveat list
- severity estimate
- missing questions to ask the vendor

## Use case 3: Reusing prior patterns for a new design

### Situation
There is a known-good prior implementation and a new target problem that is similar but not identical.

### Useful workflows
- Pattern Remix
- pattern-remix-planner

### Desired output
- draft plan
- design structure
- explicit constraints
- known risks

## Use case 4: Big repo orientation

### Situation
A repo is too large for one clean reasoning pass and the user wants fast orientation plus a path to deeper analysis.

### Useful workflows
- Context Sharding
- doc-indexer
- doc-reader
- synthesis-brief-writer

### Desired output
- repo map
- relevant modules
- likely complexity areas
- next inspection targets
