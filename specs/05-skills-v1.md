# V1 Skills

## 1. pattern-remix.md

### When to use

When you have prior successful work and need a strong first draft for a new but related problem.

### Inputs

- prior artifacts
- target state
- constraints
- anti-patterns

### Expected output

- draft plan or proposal
- explicit assumptions
- known risks

## 2. complexity-mapper.md

### When to use

When a design looks viable on paper but may hide operational, networking, cost, or implementation complexity.

### Inputs

- docs
- design assumptions
- constraints
- operational context

### Expected output

- hidden complexity summary
- risk list
- blow-up points
- missing questions

## 3. context-sharding.md

### When to use

When the source material is too large or too varied for a single clean reasoning session.

### Inputs

- large docs
- many docs
- big codebases

### Expected output

- work split plan
- distilled packets
- source maps

## 4. decision-brief.md

### When to use

When findings need to be presented to engineers, managers, or review bodies in a compact and useful format.

### Inputs

- extracted findings
- ranked risks
- assumptions

### Expected output

- concise decision brief
- evidence vs inference separation
- unresolved questions

## 5. architecture-risk-review.md

### When to use

When you want a targeted review of failure modes, hidden dependencies, limits, and operational burden.

### Inputs

- design artifacts
- caveats
- dependency map

### Expected output

- risk review summary
- severity areas
- follow-up actions
