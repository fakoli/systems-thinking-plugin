# Reference Prompts

Effective prompts and prompting patterns that have worked well in past analysis sessions.

## What goes here

- Prompts that surfaced hidden constraints or assumptions
- Prompting patterns that produced useful tradeoff analysis
- Framing techniques that forced ambiguity into the open
- Prompts that set the right level of analysis depth or tone

## How agents and skills use this folder

These files help Claude infer your preferred working style, analysis depth, and tone when running any skill. All skills can reference these to calibrate their approach — the patterns here act as style and rigor signals across the plugin.

## File guidelines

- One prompt pattern per file.
- Include context about when and why it worked well (problem type, what it surfaced).
- Raw prompts are fine. Light annotation explaining the intent is better.

## Example filenames

- `constraint-surfacing-prompt.md`
- `tradeoff-analysis-prompt.md`
- `assumption-forcing-prompt.md`
